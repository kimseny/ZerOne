# 곁다리 실습 — `recv()` 하나로는 왜 파일이 깨지나

윗 폴더의 진짜 `server.py` / `client.py` 는 이미 `makefile` 로 문제를 **해결한** 버전입니다.
그래서 정작 "한 줄이 잘리는 고통"을 겪을 수가 없습니다.
이 폴더는 그 고통을 **일부러 재현**해서, 왜 `makefile` 과 크기 제한이 필요한지 몸으로 느끼게 합니다.

## 파일
| 파일 | 설명 |
|------|------|
| `server_broken.py` | `recv(1024)` 로 받는 **고장난** 서버 + **크기 제한 없음(무조건 허용)** |
| `client_broken.py` | 받기도 `recv(1024)`. 받은 파일은 `downloads_broken/` 에 저장 |
| `small.png` | 작은 파일 (base64 100글자) — 어쩌다 **멀쩡히** 도착 |
| `big.png` | 큰 파일 (base64 36만 글자) — recv 경계에서 **잘려 깨짐** |

## 실행
```bash
# 터미널 1 — 고장난 서버 (포트 5001, 진짜 서버 5000과 안 겹침)
python server_broken.py

# 터미널 2, 3 — 고장난 클라이언트
python client_broken.py
```

## 무엇을 관찰하나 (수업 흐름)

### ① 작은 파일 — "어? 되는데?"  (함정)
```
/file small.png
```
- 받는 쪽 `downloads_broken/` 에 저장된 `small.png` 를 **열어보면 멀쩡**하다.
- 이유: base64 가 100글자뿐이라 `recv(1024)` **한 번에 통째로** 들어왔다.
- 🪤 함정: "내 컴퓨터에선 되던데?" — 이게 이런 버그가 무서운 이유다.

### ② 큰 파일 — 같은 코드인데 깨진다
```
/file big.png
```
- 저장된 `big.png` 를 열어보면 **깨져서 안 열리거나 반쪽**만 보인다.
- 이유: base64 가 36만 글자인데 `recv(1024)` 는 **1024바이트까지만** 준다.
  나머지는 다음 recv 로 오는데, 그 뒷토막은 `TEXT|`·`FILE|` 로 시작하지 않아
  서버 화면에 `(알 수 없는 종류 = 잘린 뒷토막?)` 으로 쏟아진다.
- 원본 270,448 bytes → 저장은 겨우 **수백 bytes**. (실측: 750 bytes)

> TCP 는 '메시지(한 줄)' 단위가 아니라 **'바이트 흐름(stream)'** 이다.
> `recv(n)` 은 "딱 한 줄"이 아니라 "그 순간 도착한 만큼(최대 n)"을 준다.

### ③ 어떻게 고치나 — 프레이밍(framing)
윗 폴더의 진짜 코드가 정답이다. `recv` 를 `makefile` 로 바꾸고 **`\n` 이 나올 때까지 모아** 한 줄을 완성한다.

| 고장난 버전 (이 폴더) | 고친 버전 (윗 폴더) |
|----------------------|---------------------|
| `chunk = conn.recv(1024)` | `reader = conn.makefile("r", ...)` |
| 받은 만큼 바로 처리 | `line = reader.readline()`  ← `\n` 까지 모아 한 줄 |

→ 고친 버전에선 `small.png` 든 `big.png` 든 **크기와 무관하게** 멀쩡히 도착한다.

### ④ 그런데 무한정 받아도 되나? — 크기 제한
프레이밍을 고쳐서 "안 깨지게" 만들어도, **아무 크기나 다 받는 건 별개의 문제**다.
- 💾 **서버 디스크**: 100명이 각자 500MB 영상을 올리면? 서버가 그걸 다 중계·보관한다.
- 🌐 **네트워크 트래픽**: base64 는 원본보다 **약 33% 더 크다**(3바이트 → 4글자).
  큰 파일일수록 회선을 오래 점유하고, 그동안 다른 사람 메시지가 밀린다.

그래서 진짜 `server.py` 에는 크기 상한이 있다 — [`../server.py`](../server.py) 의
`MAX_FILE_BYTES = 5 * 1024 * 1024`(5MB). 한도를 넘으면 **받기 전에 거절**한다.
이 폴더의 `server_broken.py` 에는 이 검사가 **일부러 빠져 있다**(무조건 허용).

## 이번 실습의 한 줄 정리
> **①②** 흐름은 잘린다 → **③** `\n` 까지 모아 읽어(프레이밍) 해결,
> 그리고 **④** 그래도 크기는 따로 제한해야 한다(디스크·트래픽).
